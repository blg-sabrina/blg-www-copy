## 💡 Bahagian 4: Konsep Teras

### Memahami Seni Bina BigLedger

{{< callout type="info" >}}
**Konsep Utama**: BigLedger menggunakan seni bina modular di mana **Applet** ialah komponen berfungsi yang boleh digunakan merentasi pelbagai **Modul** (bidang perniagaan).
{{< /callout >}}

### Terminologi Penting

| Istilah | Definisi | Contoh |
|------|------------|---------|
| **Tenant** | Contoh syarikat anda | demo-v1 |
| **Module** | Bidang berfungsi perniagaan | Jualan, Inventori |
| **Applet** | Komponen boleh guna semula | Konfigurasi Cukai |
| **Document** | Transaksi perniagaan | Invois, PO |
| **Workflow** | Proses automatik | Rantaian kelulusan |
| **Master Data** | Data perniagaan teras | Pelanggan, Item |

## 🚀 Bahagian 5: Transaksi Pertama Anda

### Mencipta Pesanan Jualan Sampel

{{< tabs items="Langkah 1: Navigasi,Langkah 2: Cipta,Langkah 3: Proses,Langkah 4: Sahkan" >}}
  {{< tab >}}
  **Menuju ke Pesanan Jualan**

  1. Klik **Sales** dalam menu utama
  2. Pilih **Sales Orders**
  3. Anda akan melihat senarai pesanan sedia ada
  4. Klik butang **New Sales Order** (kanan atas)
  {{< /tab >}}

  {{< tab >}}
  **Masukkan Butiran Pesanan**

  1. **Bahagian Pelanggan**
     - Cari: Taip "ACME"
     - Pilih: "ACME Corporation"
     - Perhatikan alamat diisi secara automatik

  2. **Tambah Item Baris**
     - Klik "Add Item"
     - Cari: "PROD-001"
     - Kuantiti: 10
     - Harga diisi secara automatik

  3. **Maklumat Pesanan**
     - Tarikh penghantaran: Esok
     - Terma pembayaran: Net 30
     - Nota: "Demo order"
  {{< /tab >}}

  {{< tab >}}
  **Simpan dan Proses**

  1. Klik **Save Draft**
     - Nombor pesanan dijana
     - Status menunjukkan "Draft"

  2. Klik **Submit for Approval**
     - Status bertukar kepada "Pending"
     - Pemberitahuan dihantar

  3. Klik **Approve** (sebagai admin)
     - Status: "Approved"
     - Sedia untuk pemenuhan
  {{< /tab >}}

  {{< tab >}}
  **Sahkan Kerja Anda**

  1. **Semak Senarai Pesanan**
     - Pesanan anda muncul di bahagian atas
     - Status ditunjukkan dengan betul

  2. **Lihat Butiran Pesanan**
     - Klik nombor pesanan
     - Semak semua maklumat

  3. **Semak Log Aktiviti**
     - Lihat cap masa penciptaan
     - Semak sejarah kelulusan
  {{< /tab >}}
{{< /tabs >}}

## 📊 Bahagian 6: Pelaporan Asas

### Menjana Laporan Pertama Anda

1. **Navigasi ke Reports**
   - Klik "Reports" dalam menu utama
   - Pilih "Sales Reports"

   - Pilih "Sales Summary"
   - Tetapkan julat tarikh: Bulan Ini

3. **Konfigurasikan Pilihan**
   - Kumpulkan mengikut: Pelanggan
   - Sertakan: Semua produk
   - Format: PDF

4. **Jana & Eksport**
   - Klik "Generate Report"
   - Pratonton keputusan
   - Eksport mengikut keperluan

## 💡 Petua untuk Kejayaan Demo

### Amalan Terbaik

{{< callout type="success" >}}
**Petua Kejayaan Demo**:
1. **Mulakan mudah** - Kuasai asas sebelum ciri lanjutan
2. **Gunakan data sampel** - Jangan risau tentang membuat kesilapan
3. **Terokai bebas** - Klik ke sana ke mari dan temui ciri
4. **Ambil nota** - Dokumenkan soalan untuk pasukan jualan
5. **Cuba aliran kerja** - Lengkapkan proses hujung ke hujung
{{< /callout >}}

### Soalan Lazim

**S: Bolehkah saya merosakkan apa-apa dalam demo?**
J: Tidak! Demo ditetapkan semula setiap hari, jadi bereksperimen dengan bebas.

**S: Bagaimana saya membuat asal perubahan?**
J: Kebanyakan tindakan mempunyai pilihan "undo" atau "reverse". Semak menu tindakan.

**S: Di manakah saya boleh mencari bantuan?**
J: Klik ikon (?) pada mana-mana halaman atau tekan F1 untuk bantuan kontekstual.
