تحتوي الوحدة الأساسية على التطبيقات الجوهرية التي تحتاجها كل الوحدات الأخرى في BigLedger. توفر هذه التطبيقات البنية التحتية الأساسية وقدرات إدارة البيانات الرئيسية التي يعتمد عليها النظام بأكمله.

## نظرة عامة

الوحدة الأساسية هي أساس نظام BigLedger لتخطيط موارد المؤسسات، وتوفر:
- **إدارة البيانات الرئيسية** - إدارة مركزية لبيانات الأعمال الحيوية
- **تهيئة النظام** - الإعدادات والتهيئات الضرورية
- **العمليات الأساسية** - عمليات الأعمال الجوهرية
- **تكامل البيانات** - بيانات مشتركة عبر كل الوحدات

{{< callout type="info" >}}
**مهم**: تطبيقات الوحدة الأساسية شرط مسبق لكل الوحدات الأخرى. يجب تهيئتها بشكل صحيح قبل تنفيذ وحدات الأعمال الأخرى.
{{< /callout >}}

## التطبيقات الأساسية

### 1. تطبيق دليل الحسابات
**الغرض**: تعريف وإدارة هيكل دليل الحسابات الكامل
- فئات الحسابات وأنواعها
- تسلسلات الحسابات
- ربط القوائم المالية
- دعم تعدد العملات

**يستخدمه**: كل المعاملات المالية عبر جميع الوحدات

### 2. تطبيق إدارة المستأجر
**الغرض**: إدارة وتهيئة على مستوى النظام كله
- إدارة المستخدمين والصلاحيات
- إعدادات النظام
- تهيئة الأمان
- إعدادات التدقيق

**يستخدمه**: مديرو النظام لإدارة النظام بشكل عام

### 3. تطبيق المؤسسة
**الغرض**: تعريف الهيكل التنظيمي
- إعداد الشركة
- تهيئة الفروع
- هيكل الأقسام
- إدارة مراكز التكلفة

**يستخدمه**: كل الوحدات للتسلسل التنظيمي

### 4. تطبيق دفتر النقدية
**الغرض**: إدارة حسابات النقد والبنوك
- إعداد حسابات النقد
- تهيئة الحسابات البنكية
- طرق الدفع
- تتبع التدفق النقدي

**يستخدمه**: كل الوحدات التي تعالج معاملات مالية

### 5. تطبيق صيانة أصناف المستندات
**الغرض**: إدارة الأصناف والخدمات القائمة على المستندات
- أصناف الخدمات
- الأصناف غير المخزنية
- قوالب المستندات
- بنود الفوترة

**يستخدمه**: العمليات القائمة على الخدمات والخدمات المهنية

### 6. تطبيق تهيئة الضرائب
**الغرض**: إعداد وإدارة الضرائب بشكل كامل
- تهيئة GST/SST
- رموز الضرائب ومعدلاتها
- المجموعات الضريبية
- قواعد الامتثال

**يستخدمه**: كل الوحدات المولّدة لمعاملات خاضعة للضريبة

### 7. تطبيق صيانة أصناف المخزون
**الغرض**: البيانات الرئيسية للمنتجات والمخزون
- إنشاء المنتجات
- فئات الأصناف
- وحدات القياس
- إعدادات المخزون

**يستخدمه**: كل الوحدات المتعلقة بالمخزون

### 8. تطبيق صيانة العملاء
**الغرض**: إدارة البيانات الرئيسية للعملاء
- ملفات العملاء
- حدود الائتمان
- شروط الدفع
- فئات العملاء

**يستخدمه**: وحدات المبيعات وCRM والذمم المدينة والوحدات الموجهة للعملاء

### 9. تطبيق صيانة الموردين
**الغرض**: البيانات الرئيسية للموردين
- ملفات الموردين
- شروط الدفع
- اتفاقيات الشراء
- فئات الموردين

**يستخدمه**: وحدات المشتريات والذمم الدائنة والتوريد

### 10. تطبيق صيانة الموظفين
**الغرض**: السجلات الرئيسية للموظفين
- ملفات الموظفين
- تعيينات الأقسام
- الأدوار والصلاحيات
- بيانات التعويضات

**يستخدمه**: وحدات الموارد البشرية والرواتب والوحدات المتعلقة بالموظفين

## الاعتمادات

### اعتمادات الوحدات
تعتمد كل وحدات BigLedger على تطبيقات الوحدة الأساسية:

| الوحدة | الاعتمادات الأساسية |
|--------|------------------|
| **المحاسبة المالية** | دليل الحسابات، تهيئة الضرائب، دفتر النقدية |
| **المبيعات وإدارة علاقات العملاء** | صيانة العملاء، تهيئة الضرائب، أصناف المستندات |
| **المشتريات** | صيانة الموردين، تهيئة الضرائب |
| **المخزون** | صيانة أصناف المخزون، المؤسسة |
| **الموارد البشرية والرواتب** | صيانة الموظفين، المؤسسة |
| **نقاط البيع** | العملاء، أصناف المخزون، دفتر النقدية، الضرائب |
| **التجارة الإلكترونية** | العملاء، أصناف المخزون، الضرائب، المؤسسة |

### تسلسل الإعداد
ترتيب التهيئة الموصى به:

1. **إعداد المؤسسة**
   - تفاصيل الشركة
   - هيكل الفروع
   - الأقسام

2. **الأساس المالي**
   - دليل الحسابات
   - تهيئة الضرائب
   - إعداد دفتر النقدية

3. **البيانات الرئيسية**
   - العملاء
   - الموردون
   - الموظفون
   - الأصناف (مخزون/مستندات)

4. **تهيئة النظام**
   - إعدادات إدارة المستأجر
   - صلاحيات المستخدمين
   - قواعد تدفقات العمل

## أفضل ممارسات التهيئة

### الإعداد الأولي
1. **خطط لهيكلك** قبل التهيئة
2. **استخدم ترميزًا متسقًا** عبر كل البيانات الرئيسية
3. **جهّز بيانات اختبار** قبل الانطلاق الفعلي
4. **وثّق التهيئات** للرجوع إليها

### اصطلاحات التسمية
- **الحسابات**: استخدم ترقيمًا منظمًا (1000-1999 الأصول، 2000-2999 الالتزامات)
- **الأصناف**: رموز قائمة على الفئات (ELEC-001، FURN-001)
- **العملاء/الموردون**: رموز منهجية (CUS-001، SUP-001)
- **الموظفون**: رموز حسب القسم (HR-001، FIN-001)

### سلامة البيانات
- فعّل **سجلات التدقيق** منذ البداية
- أنشئ **تدفقات اعتماد** لتغييرات البيانات الرئيسية
- إجراء فحوص **تحقق من البيانات** بانتظام
- طبّق إجراءات **ضبط التغييرات**

## نقاط التكامل

### تدفق البيانات عبر الوحدات

```
الوحدة الأساسية (البيانات الرئيسية) → المحاسبة المالية | المبيعات وCRM | المخزون والمستودعات | الموارد البشرية والرواتب → بيانات المعاملات (الطلبات والفواتير وغيرها)
```

### تكامل API
توفر تطبيقات الوحدة الأساسية واجهات API من أجل:
- مزامنة البيانات الرئيسية
- التحقق اللحظي
- تصدير/استيراد البيانات
- تكامل الأطراف الخارجية

## حالات الاستخدام الشائعة

### عمليات متعددة الفروع
- دليل حسابات مركزي
- دفاتر نقدية خاصة بكل فرع
- تقارير موحدة
- معاملات بين الفروع

### أعمال متعددة العملات
- تهيئة العملات في المؤسسة
- أسعار الصرف في الإعدادات المالية
- تعدد العملات في العملاء/الموردين
- تسعير خاص بكل عملة

### متطلبات الامتثال
- إعداد GST/SST في تهيئة الضرائب
- سجلات التدقيق في إدارة المستأجر
- التقارير القانونية من دليل الحسابات
- عمليات التحقق من الامتثال

## استكشاف الأخطاء

### مشكلات شائعة

**تعذر إنشاء المعاملات**
- تحقق من اكتمال دليل الحسابات
- افحص تهيئة الضرائب
- تأكد من إعداد دفتر النقدية

**البيانات الرئيسية لا تظهر**
- افحص هيكل المؤسسة
- تحقق من صلاحيات المستخدمين
- تأكد من حالة البيانات (نشط/غير نشط)

**إخفاقات التكامل**
- تحقق من اكتمال البيانات الرئيسية
- افحص ربط الحقول
- راجع صلاحيات API

## الخطوات التالية

بعد تهيئة تطبيقات الوحدة الأساسية:

1. **أعدّ المحاسبة المالية** للعمليات المالية
2. **هيّئ المبيعات/المشتريات** للعمليات التجارية
3. **نفّذ المخزون** لإدارة المخزون
4. **أضف الوحدات المتخصصة** وفق احتياجات العمل
