## 📚 الجزء 1: أول تسجيل دخول والتنقل

### الخطوة 1: الوصول إلى البيئة التجريبية

{{< tabs items="عملية تسجيل الدخول,أساسيات التنقل,جولة في الواجهة" >}}
  {{< tab >}}
  **1. افتح الموقع التجريبي**
  - انتقل إلى: https://demo-v1.bigledger.com
  - سترى شاشة تسجيل الدخول إلى BigLedger

  **2. أدخل بيانات الاعتماد**
  - اسم المستخدم: `demo-admin`
  - كلمة المرور: `Demo2025!`
  - انقر على "تسجيل الدخول"

  **3. نجاح أول تسجيل دخول**
  - تُحمّل لوحة المعلومات مع بيانات نموذجية
  - لاحظ القائمة الرئيسية على اليسار
  - يعرض الشريط العلوي الإشعارات والملف الشخصي للمستخدم
  {{< /tab >}}

  {{< tab >}}
  **بنية القائمة الرئيسية**
  - **لوحة المعلومات**: الشاشة الرئيسية مع مؤشرات الأداء الرئيسية
  - **المبيعات**: إدارة العملاء والطلبات
  - **المشتريات**: الموردون والتوريد
  - **المخزون**: إدارة المخزون
  - **المحاسبة**: العمليات المالية
  - **التقارير**: التحليلات والرؤى
  - **الإعدادات**: تكوين النظام

  **نصائح التنقل**
  - انقر على عناصر القائمة لتوسيع القوائم الفرعية
  - استخدم مسارات التنقل لتتبع موقعك
  - ضع علامة نجمة على الصفحات المستخدمة بشكل متكرر
  {{< /tab >}}

  {{< tab >}}
  **عناصر الواجهة الرئيسية**

  1. **شريط التنقل العلوي**
     - محدد الشركة (في حالة تعدد الشركات)
     - البحث السريع (Ctrl+K)
     - جرس الإشعارات
     - قائمة الملف الشخصي للمستخدم

  2. **الشريط الجانبي الأيسر**
     - القائمة الرئيسية القابلة للطي
     - التنقل بين الوحدات
     - المفضلة للوصول السريع

  3. **منطقة المحتوى الرئيسية**
     - عنوان الصفحة والإجراءات
     - شبكات البيانات والنماذج
     - التنقل بين علامات التبويب للبيانات ذات الصلة

  4. **أزرار الإجراءات**
     - الإجراءات الأساسية (أزرق)
     - الإجراءات الثانوية (رمادي)
     - إجراءات الخطر (أحمر)
  {{< /tab >}}
{{< /tabs >}}

## 🔧 الجزء 3: التكوين الأساسي

### الخطوة 2: اضبط تفضيلاتك

{{< tabs items="إعدادات المستخدم,خيارات العرض,الإشعارات" >}}
  {{< tab >}}
  **انتقل إلى: الإعدادات > ملفي الشخصي**

  1. **المعلومات الشخصية**
     - تحديث اسم العرض
     - تعيين صورة الملف الشخصي
     - تكوين معلومات الاتصال

  2. **الإعدادات الإقليمية**
     - اللغة: الإنجليزية
     - تنسيق التاريخ: MM/DD/YYYY
     - عرض العملة: USD
     - تنسيق الأرقام: 1,234.56

  3. **حفظ التغييرات**
     - انقر على "حفظ الملف الشخصي"
     - تُطبق التغييرات فورًا
  {{< /tab >}}

  {{< tab >}}
  **انتقل إلى: الإعدادات > العرض**

  1. **اختيار السمة**
     - الوضع الفاتح (افتراضي)
     - الوضع الداكن
     - تلقائي (يتبع النظام)

  2. **خيارات التخطيط**
     - عرض مضغوط
     - عرض مريح
     - إعدادات كثافة الشبكة

  3. **تفضيلات القوائم**
     - السجلات لكل صفحة: 25
     - ترتيب الفرز الافتراضي
     - رؤية الأعمدة
  {{< /tab >}}

  {{< tab >}}
  **انتقل إلى: الإعدادات > الإشعارات**

  1. **تنبيهات البريد الإلكتروني**
     - تأكيدات الطلبات
     - تحذيرات انخفاض المخزون
     - تذكيرات الدفع

  2. **الإشعارات داخل التطبيق**
     - تعيينات المهام
     - طلبات الموافقة
     - تحديثات النظام

  3. **إعدادات التكرار**
     - في الوقت الفعلي
     - ملخص يومي
     - ملخص أسبوعي
  {{< /tab >}}

## 💡 الجزء 4: المفاهيم الأساسية

### فهم بنية BigLedger

{{< callout type="info" >}}
**مفهوم رئيسي**: يستخدم BigLedger بنية معيارية حيث تكون **التطبيقات المصغّرة (Applets)** مكونات وظيفية يمكن استخدامها عبر عدة **وحدات (Modules)** (مجالات الأعمال).
{{< /callout >}}

### المصطلحات الأساسية

| المصطلح | التعريف | مثال |
|------|------------|---------|
| **المستأجر (Tenant)** | مثيل شركتك | demo-v1 |
| **الوحدة (Module)** | مجال وظيفي للأعمال | المبيعات، المخزون |
| **التطبيق المصغّر (Applet)** | مكون قابل لإعادة الاستخدام | تكوين الضرائب |
| **المستند (Document)** | معاملة تجارية | فاتورة، أمر شراء |
| **سير العمل (Workflow)** | عملية آلية | سلسلة الموافقات |
| **البيانات الرئيسية (Master Data)** | بيانات الأعمال الأساسية | العملاء، الأصناف |

## 🚀 الجزء 5: أول معاملة لك

### إنشاء أمر مبيعات نموذجي

{{< tabs items="الخطوة 1: التنقل,الخطوة 2: الإنشاء,الخطوة 3: المعالجة,الخطوة 4: التحقق" >}}
  {{< tab >}}
  **الوصول إلى أوامر المبيعات**

  1. انقر على **المبيعات** في القائمة الرئيسية
  2. اختر **أوامر المبيعات**
  3. سترى قائمة الأوامر الموجودة
  4. انقر على زر **أمر مبيعات جديد** (أعلى اليمين)
  {{< /tab >}}

  {{< tab >}}
  **إدخال تفاصيل الأمر**

  1. **قسم العميل**
     - البحث: اكتب "ACME"
     - اختر: "ACME Corporation"
     - لاحظ التعبئة التلقائية للعنوان

  2. **إضافة بنود السطر**
     - انقر على "إضافة صنف"
     - البحث: "PROD-001"
     - الكمية: 10
     - يُملأ السعر تلقائيًا

  3. **معلومات الأمر**
     - تاريخ التسليم: غدًا
     - شروط الدفع: صافي 30 يومًا
     - ملاحظات: "أمر تجريبي"
  {{< /tab >}}

  {{< tab >}}
  **الحفظ والمعالجة**

  1. انقر على **حفظ كمسودة**
     - يُنشأ رقم الأمر
     - تُظهر الحالة "مسودة"

  2. انقر على **إرسال للموافقة**
     - تتغير الحالة إلى "قيد الانتظار"
     - يُرسل إشعار

  3. انقر على **موافقة** (كمسؤول)
     - الحالة: "تمت الموافقة"
     - جاهز للتنفيذ
  {{< /tab >}}

  {{< tab >}}
  **تحقق من عملك**

  1. **تحقق من قائمة الأوامر**
     - يظهر أمرك في الأعلى
     - تُعرض الحالة بشكل صحيح

  2. **عرض تفاصيل الأمر**
     - انقر على رقم الأمر
     - راجع جميع المعلومات

  3. **تحقق من سجل النشاط**
     - اطلع على طابع الوقت للإنشاء
     - راجع سجل الموافقات
  {{< /tab >}}
{{< /tabs >}}

## 📊 الجزء 6: إعداد التقارير الأساسية

### إنشاء أول تقرير لك

1. **انتقل إلى التقارير**
   - انقر على "التقارير" في القائمة الرئيسية
   - اختر "تقارير المبيعات"

2. **اختر نوع التقرير**
   - اختر "ملخص المبيعات"
   - عيّن النطاق الزمني: هذا الشهر

3. **تكوين الخيارات**
   - التجميع حسب: العميل
   - التضمين: جميع المنتجات
   - التنسيق: PDF

4. **الإنشاء والتصدير**
   - انقر على "إنشاء التقرير"
   - عاين النتائج
   - صدّر حسب الحاجة

## 💡 نصائح لنجاح النسخة التجريبية

### أفضل الممارسات

{{< callout type="success" >}}
**نصائح لنجاح النسخة التجريبية**:
1. **ابدأ بالبساطة** - أتقن الأساسيات قبل الميزات المتقدمة
2. **استخدم بيانات نموذجية** - لا تقلق بشأن ارتكاب الأخطاء
3. **استكشف بحرية** - انقر في أنحاء المنصة واكتشف الميزات
4. **دوّن الملاحظات** - وثّق الأسئلة لفريق المبيعات
5. **جرّب سير العمل** - أكمل العمليات من البداية إلى النهاية
{{< /callout >}}

### الأسئلة الشائعة

**س: هل يمكنني إتلاف أي شيء في النسخة التجريبية؟**
ج: لا! تُعاد النسخة التجريبية يوميًا، لذا جرّب بحرية.

**س: كيف أتراجع عن التغييرات؟**
ج: تحتوي معظم الإجراءات على خيار "تراجع" أو "عكس". تحقق من قائمة الإجراءات.

**س: أين يمكنني العثور على المساعدة؟**
ج: انقر على أيقونة (?) في أي صفحة أو اضغط F1 للمساعدة السياقية.
