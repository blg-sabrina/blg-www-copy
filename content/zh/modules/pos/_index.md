销售点（POS）模块提供全面的零售门店运营能力，实现流畅的收银操作、多门店管理以及稳健的离线/在线数据同步。

## 概述

POS 模块专为有以下需求的零售企业设计：
- **店内销售处理** - 快速可靠的销售点交易
- **多门店管理** - 对多个零售网点的集中管控
- **离线能力** - 即使断网也能继续营业
- **实时同步** - 联网后自动同步数据
- **全面报表** - 销售分析与门店经营洞察

{{< callout type="info" >}}
**模块依赖**：需先配置核心模块应用（客户维护、库存商品维护、税务配置、现金簿、组织）。
{{< /callout >}}

## 业务流程

### 标准销售交易
```
顾客将商品带到收银台 → 收银员扫描商品（POS 终端）→ 系统含税计算总额 → 顾客付款（现金管理）→ 打印小票（小票管理）→ 交易同步至总部
```

### 日终流程
```
完成所有待处理交易 → 盘点钱箱（现金管理）→ 生成每日报表（POS 报表）→ 核对付款与存款 → 数据同步至中央系统
```

### 离线运营
```
系统检测到网络断开 → 切换至离线模式（POS 离线同步）→ 本地存储交易 → 继续正常营业 → 网络恢复后自动同步
```

## 实施场景

### 单店部署
适用于：
- 小型零售企业
- 精品店
- 专卖店
- 服务柜台

**主要特性**：
- 简单的部署流程
- 基础报表
- 核心 POS 功能
- 本地客户数据库

### 多店连锁
非常适合：
- 零售连锁
- 加盟经营
- 百货商场
- 购物中心网点

**主要特性**：
- 集中式库存管理
- 门店间调拨
- 合并报表
- 全连锁促销

### 离线优先零售
专为以下场景设计：
- 偏远地区
- 集市与展会
- 移动销售单元
- 网络条件差的区域

**主要特性**：
- 稳健的离线能力
- 批量同步
- 冲突化解
- 数据完整性保护

## 部署要求

### 硬件要求
1. **POS 终端/电脑**
   - 触屏能力
   - 至少 4GB 内存
   - 用于离线运营的本地存储

2. **外设**
   - 条码扫描枪
   - 小票打印机
   - 钱箱
   - 客显屏（可选）

3. **网络部署**
   - 可靠的互联网连接
   - 多终端本地网络
   - 备用连接（4G/移动网络）

### 软件前置条件
1. **核心模块配置**
   - 组织结构
   - 会计科目表
   - 税务配置
   - 客户与商品主数据

2. **POS 专项设置**
   - 门店配置
   - 终端分配
   - 小票模板
   - 支付方式

## 故障排查

### 常见问题

**POS 终端无响应**
- 检查网络连接
- 重启 POS 应用
- 核实硬件连接
- 检查系统资源

**离线同步问题**
- 核实互联网连接
- 检查同步服务状态
- 查看错误日志
- 联系 IT 支持

**打印机问题**
- 检查打印机连接
- 核实纸张与墨量
- 更新打印机驱动
- 换用其他小票模板测试
