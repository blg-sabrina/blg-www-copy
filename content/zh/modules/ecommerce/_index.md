电商模块帮助企业通过多渠道开展线上销售，包括自有网站、电商平台与 B2B 门户，并实现线上与线下运营的无缝衔接。

## 模块概述

电商模块由多个专业应用组成，协同提供：
- **多渠道销售** - 覆盖各大电商平台与自有网站
- **库存同步** - 线上线下库存联动
- **订单管理** - 汇聚多来源订单
- **网站管理** - 基于 CP-Commerce
- **B2B 业务** - 面向批发客户

## 核心依赖

本模块需要以下核心模块应用：
- **客户维护** - 客户数据与账户
- **库存商品维护** - 商品主数据
- **税务配置** - 线上销售的税费计算
- **组织** - 多地点库存
- **现金簿** - 收款处理

## 应用配置

### 设置顺序

1. **核心模块设置**
   - 在库存商品维护中配置商品
   - 在客户维护中建立客户
   - 配置税码

2. **EcomSync 配置**
   - 连接平台账号
   - 映射商品目录
   - 设置同步规则

3. **CP-Commerce 设置**（如使用自有网站）
   - 配置网站设置
   - 设计站点结构
   - 设置支付网关

4. **渠道激活**
   - 启用各电商平台
   - 配置渠道专属设置
   - 测试订单流程

## 常见工作流

### 商品上架
1. 在库存商品维护中创建商品
2. 在 EcomSync 中配置
3. 推送到选定渠道
4. 监控上架状态

### 订单履约
1. 订单从各渠道自动导入
2. 自动预留库存
3. 执行拣货与打包
4. 更新发货状态
5. 将物流单号同步至平台

### 库存更新
1. BigLedger 中库存变动
2. EcomSync 计算可售数量
3. 更新推送至所有渠道
4. 防止超卖

## 故障排查

### 常见问题

**商品未同步**
- 核实平台授权
- 检查商品映射
- 查看同步日志
- 校验必填字段

**订单导入失败**
- 检查 API 连接
- 核实客户数据
- 复核税务配置
- 检查库存可用性

**库存不一致**
- 执行库存对账
- 检查同步频率
- 复核缓冲库存设置
- 核实多地点配置
