核心模块包含 BigLedger 中所有其他模块都需要的基础应用。这些应用提供整个系统所依赖的基础架构与主数据管理能力。

## 概述

核心模块是 BigLedger ERP 的基石，提供：
- **主数据管理** - 集中管理关键业务数据
- **系统配置** - 必要的设置与配置
- **基础操作** - 基本业务操作
- **数据集成** - 跨所有模块共享数据

{{< callout type="info" >}}
**重要**：核心模块应用是所有其他模块的前置条件。在实施其他业务模块之前，必须先正确完成核心模块应用的配置。
{{< /callout >}}

## 核心应用

### 1. 会计科目表应用
**用途**：定义并管理完整的会计科目表结构
- 科目类别与类型
- 科目层级
- 财务报表映射
- 多币种支持

**使用方**：所有模块的全部财务交易

### 2. 租户管理应用
**用途**：全系统的管理与配置
- 用户管理与权限
- 系统设置
- 安全配置
- 审计设置

**使用方**：系统管理员进行整体系统管理

### 3. 组织应用
**用途**：定义组织结构
- 公司设置
- 分支机构配置
- 部门结构
- 成本中心管理

**使用方**：所有模块的组织层级

### 4. 现金簿应用
**用途**：现金与银行账户管理
- 现金账户设置
- 银行账户配置
- 付款方式
- 现金流追踪

**使用方**：所有处理财务交易的模块

### 5. 单据商品维护应用
**用途**：管理基于单据的商品与服务
- 服务类商品
- 非库存商品
- 单据模板
- 计费项目

**使用方**：服务型业务、专业服务

### 6. 税务配置应用
**用途**：完整的税务设置与管理
- GST/SST 配置
- 税码与税率
- 税组
- 合规规则

**使用方**：所有产生应税交易的模块

### 7. 库存商品维护应用
**用途**：产品与库存主数据
- 产品创建
- 商品类别
- 计量单位
- 库存设置

**使用方**：所有与库存相关的模块

### 8. 客户维护应用
**用途**：客户主数据管理
- 客户档案
- 信用额度
- 付款条款
- 客户类别

**使用方**：销售、CRM、应收账款及面向客户的模块

### 9. 供应商维护应用
**用途**：供应商主数据
- 供应商档案
- 付款条款
- 采购协议
- 供应商类别

**使用方**：采购、应付账款及采购管理模块

### 10. 员工维护应用
**用途**：员工主记录
- 员工档案
- 部门分配
- 角色与权限
- 薪酬数据

**使用方**：HR、薪资及员工相关模块

## 依赖关系

### 模块依赖
所有 BigLedger 模块都依赖核心模块应用：

| 模块 | 核心依赖 |
|--------|------------------|
| **财务会计** | 会计科目表、税务配置、现金簿 |
| **销售与CRM** | 客户维护、税务配置、单据商品 |
| **采购** | 供应商维护、税务配置 |
| **库存** | 库存商品维护、组织 |
| **HR 与薪资** | 员工维护、组织 |
| **POS** | 客户、库存商品、现金簿、税务 |
| **电商** | 客户、库存商品、税务、组织 |

### 设置顺序
推荐的配置顺序：

1. **组织设置**
   - 公司信息
   - 分支结构
   - 部门

2. **财务基础**
   - 会计科目表
   - 税务配置
   - 现金簿设置

3. **主数据**
   - 客户
   - 供应商
   - 员工
   - 商品（库存/单据）

4. **系统配置**
   - 租户管理设置
   - 用户权限
   - 工作流规则

## 配置最佳实践

### 初始设置
1. 配置前**先规划好结构**
2. 所有主数据**采用一致的编码**
3. 上线前**准备测试数据**
4. **记录配置内容**以备查阅

### 命名规范
- **科目**：采用结构化编号（1000-1999 资产，2000-2999 负债）
- **商品**：按类别编码（ELEC-001、FURN-001）
- **客户/供应商**：系统化编码（CUS-001、SUP-001）
- **员工**：按部门编码（HR-001、FIN-001）

### 数据完整性
- 从一开始就启用**审计轨迹**
- 为主数据变更建立**审批工作流**
- 定期进行**数据校验**检查
- 落地**变更控制**流程

## 集成点

### 跨模块数据流

```
核心模块（主数据） → 财务会计 | 销售与CRM | 库存与仓储 | HR 与薪资 → 交易数据（订单、发票等）
```

### API 集成
核心模块应用提供以下 API 能力：
- 主数据同步
- 实时校验
- 数据导出/导入
- 第三方集成

## 常见使用场景

### 多分支运营
- 集中式会计科目表
- 各分支专属现金簿
- 合并报表
- 分支间交易

### 多币种业务
- 在组织中配置币种
- 在财务设置中维护汇率
- 客户/供应商支持多币种
- 按币种定价

### 合规要求
- 在税务配置中设置 GST/SST
- 在租户管理中启用审计轨迹
- 基于会计科目表生成法定报表
- 合规校验

## 故障排查

### 常见问题

**无法创建交易**
- 核实会计科目表是否完整
- 检查税务配置
- 确认现金簿已设置

**主数据不显示**
- 检查组织结构
- 核实用户权限
- 确认数据状态（启用/停用）

**集成失败**
- 校验主数据的完整性
- 检查字段映射
- 检查 API 权限
