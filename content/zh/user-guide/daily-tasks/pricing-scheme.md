掌握 BigLedger 灵活的定价系统，创建精细的定价策略，以适应不同的客户、市场和业务场景，同时保持盈利能力和竞争定位。

## 概述

BigLedger 中的定价方案是强大的模板，用于定义产品和服务的定价结构。这些方案可以应用于多个业务模块，包括 POS、销售订单、报价单和电子商务平台，在提供一致定价的同时允许客户专属和基于场景的灵活性。

{{< callout type="info" >}}
**关键概念**：定价方案创建框架，而实际项目价格则在 [项目维护](/zh/user-guide/item-maintenance/) 模块中设置。这种分离允许在不同客户细分和销售渠道之间实施灵活的定价策略。
{{< /callout >}}

## 了解定价方案

### 什么是定价方案？

定价方案是定义不同类型价格如何计算和应用的模板：

- **价格类型**：零售、批发、会员、促销、最低、最高价格
- **客户细分**：为不同客户类别设置不同定价
- **渠道特定**：为 POS、电子商务、B2B 门户设置独特定价
- **时效性**：季节性、促销或基于时间的定价
- **基于数量**：数量档次和批量定价结构

### 跨模块集成

定价方案与以下模块无缝集成：
- **POS 系统**：基于客户类型的自动价格选择
- **电子商务平台**：在线渠道的动态定价
- **销售订单**：客户专属定价应用
- **报价系统**：专业的定价展示
- **采购**：成本加成定价计算

## 访问定价方案管理

### 导航与界面

1. 从主菜单导航到 **定价方案（Pricing Scheme）**
2. 访问具有高级搜索功能的全面列表页面
3. 使用 AG-Grid 界面进行高效的方案管理
4. 利用模糊搜索快速定位方案

### 列表页面功能

**搜索与筛选功能**
- **常规搜索**：在所有方案字段中快速进行基于文本的搜索
- **高级搜索**：按状态、类型、日期范围进行详细筛选
- **模糊搜索**：即使使用部分或近似术语也能找到方案
- **排序选项**：按名称、创建日期、最后修改时间、使用频率进行组织

**可视化指示器**
- **状态图标**：启用、停用、草稿方案指示器
- **使用统计**：显示哪些方案被最频繁使用
- **集成状态**：指示哪些模块正在使用每个方案
- **最后修改**：跟踪近期变更和更新

## 创建定价方案

### 方案创建流程

#### 1. 启动新方案创建

1. 在定价方案列表页面上点击 **"+" 按钮**
2. 创建对话框在辅助容器中打开
3. 填写必填和可选信息
4. 保存以创建新的定价方案模板

#### 2. 核心方案信息

**系统生成的代码**
- 由 BigLedger 自动创建
- 用于内部跟踪的唯一标识符
- 用于 API 调用和数据导出
- 创建后无法修改

**方案名称** *（必填）*
- 便于识别的描述性名称
- 最佳实践：包含用途和目标市场
- 示例："零售标准"、"批发批量"、"会员折扣"
- 保持名称一致和专业

**描述** *（可选）*
- 对方案用途和使用方式的详细说明
- 包含目标客户细分
- 注明任何特殊条件或限制
- 有助于团队理解和培训

### 方案配置示例

**零售定价方案**
- 名称："零售标准定价"
- 描述："适用于到店客户和在线销售的标准零售价格"
- 用途：POS 系统、电子商务平台、一般报价单

**批发定价方案**
- 名称："批发 B2B 定价"
- 描述："面向企业客户的带数量折扣的批量定价"
- 用途：B2B 门户、大宗订单处理、经销商销售

**会员定价方案**
- 名称："VIP 会员专享"
- 描述："面向忠诚度计划会员的带额外折扣的特殊定价"
- 用途：会员门户、带客户识别的 POS、特殊促销

## 编辑和管理定价方案

### 访问方案编辑

1. 从主列表中点击任意定价方案
2. 访问全面的编辑界面
3. 在不同配置选项卡之间导航
4. 进行更改并保存更新

### 主选项卡配置

**核心信息管理**
- **定价代码**：查看系统代码（只读参考）
- **方案名称**：更新命名以提高清晰度和一致性
- **描述**：修改详细说明和使用注释
- **状态管理**：控制启用/停用状态以控制方案可用性

**状态控制选项**
- **启用**：方案可在所有集成模块中使用
- **停用**：方案从选择中隐藏，但现有使用保留
- **草稿**：方案正在开发中，不可用于运营

**方案管理操作**
- **保存更改**：更新方案信息
- **复制方案**：为类似定价策略创建副本
- **删除方案**：删除未使用的方案（带安全检查）
- **导出设置**：生成方案配置用于备份或传输

### 复制选项卡功能

**方案复制与模板**
- **源方案选择**：选择现有方案作为起点
- **修改选项**：调整定价百分比、添加/删除价格类型
- **批量更新**：跨多个价格点应用更改
- **模板创建**：保存为可重用模板以供未来方案使用

**高级复制功能**
- **选择性复制**：选择特定元素进行复制
- **价格调整**：在复制期间应用百分比增加或减少
- **客户细分映射**：为不同市场细分调整方案
- **多币种支持**：为国际市场转换定价

## 定价方案应用

### POS 集成

**POS 配置设置**
1. 从主导航访问 POS 设置
2. 导航到定价配置部分
3. 为不同场景选择合适的定价方案：
   - **最低价格**：最低允许售价
   - **最高价格**：最高建议售价
   - **零售价格**：标准客户定价
   - **会员价格**：面向忠诚度会员的特殊定价

**动态价格选择**
- **客户类型识别**：基于客户档案自动选择方案
- **手动覆盖**：员工可在需要时选择合适的方案
- **促销集成**：在特殊活动期间应用促销方案
- **税务合规**：确保所有定价符合马来西亚 GST/SST 合规要求

### 电子商务平台集成

**多渠道定价**
- **平台特定方案**：为 Shopify、Lazada、Shopee 设置不同定价
- **货币适配**：为国际销售自动转换
- **促销定价**：营销活动的时效性定价
- **基于库存的定价**：基于库存水平的价格调整

### 销售订单和报价集成

**专业定价展示**
- **客户专属定价**：基于客户分类自动应用
- **数量折扣**：基于数量的定价计算
- **项目定价**：面向大型项目或合同的特殊方案
- **审批工作流程**：特殊定价请求的管理层审批

## 高级定价策略

### 多层级定价系统

**客户细分**
- **零售客户**：带完整毛利的标准定价
- **批发客户**：带降低毛利的基于数量的定价
- **VIP 会员**：带忠诚度折扣的特殊定价
- **员工**：员工采购的内部定价

**地理定价**
- **本地市场**：标准区域定价
- **出口市场**：考虑货币因素的国际定价
- **偏远地区**：考虑配送成本的调整定价
- **城市与农村**：市场特定的定价策略

### 基于时间的定价

**季节性调整**
- **旺季**：高需求期间的较高价格
- **淡季**：为维持销量而制定的促销定价
- **节日特惠**：活动专属定价方案
- **季末**：用于库存管理的清仓定价

**动态定价**
- **实时调整**：市场响应式的价格变化
- **竞争定价**：根据市场状况调整
- **库存驱动**：基于库存水平的价格调整
- **基于需求**：高需求商品的较高价格

### 战略规划

**市场分析**
- 研究竞争对手的定价策略
- 了解客户价格敏感度

- 确保所有方案保持可接受的利润率

- 定期审查和调整定价策略

**一致应用**
- 培训所有员工正确使用方案
- 记录定价政策和程序
- 定期审计以确保正确实施
- 就定价结构与客户沟通

**系统集成**
- 部署前在所有平台上测试定价方案
- 验证税费计算和合规要求
- 确保模块之间的数据同步
- 定期备份定价方案配置

### 监控与优化

**绩效跟踪**
- 按定价方案监控销售绩效
- 分析客户对不同价格点的反应
- 按客户细分和产品跟踪盈利能力
- 定期审查竞争定位

**持续改进**
- 定期方案审查和优化
- 客户反馈集成
- 市场趋势适应
- 技术增强采用

## 常见问题排查

### 定价不一致

**多平台同步**
- **问题**：价格未能在所有平台上更新
- **解决方案**：检查 EcomSync 配置和 API 连接
- **预防**：定期同步测试和监控

**税费计算错误**
- **问题**：定价中的税额不正确
- **解决方案**：验证税码分配和费率配置
- **预防**：定期合规审查和更新

### 性能问题

**大型目录管理**
- **问题**：项目众多时定价更新缓慢
- **解决方案**：实施批处理和计划更新
- **预防**：优化数据库索引并定期维护

**用户体验问题**
- **问题**：员工对方案选择感到困惑
- **解决方案**：提供全面培训和清晰文档
- **预防**：简化方案名称并提供使用指南
