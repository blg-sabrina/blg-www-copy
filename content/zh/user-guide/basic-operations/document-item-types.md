了解 BigLedger 的文档项目类型对于有效的库存管理、销售处理和业务运营至关重要。每种项目类型服务于特定的业务场景，并为管理复杂的库存关系提供独特的功能。

## 概述

BigLedger 支持五种不同的文档项目类型，每种类型都针对特定的业务场景和库存关系而设计。这些类型决定了项目在交易、库存跟踪和财务报告中的行为方式。

{{< callout type="info" >}}
**基础概念**：BigLedger 中的文档项目代表可以销售、采购或管理的业务实体。项目类型决定了文档项目 (FI-item) 与实物库存项目 (INV items) 之间的关系。
{{< /callout >}}

---

## 五种文档项目类型

### 1. 基本项目 (FI-Item)

**定义**：基本项目与实物库存项目 (INV item) 保持一对一关系，代表 BigLedger 中最简单、最常见的项目类型。

#### 主要特征

- **直接映射**：一个 FI-item 等于一个 INV item
- **库存跟踪**：实时库存水平更新
- **简单处理**：直接的销售和采购交易
- **财务集成**：直接的成本和收入过账
- **条形码支持**：标准条形码扫描和识别

{{< /cards >}}

#### 库存行为

- **库存扣减**：销售时立即减少库存
- **收货**：采购或收货时直接增加库存
- **估值**：单个项目成本和售价
- **报告**：单个项目跟踪和分析

### 2. 套件项目 (Kit Item)

**定义**：套件将多个基本项目组合成单个可销售单位，同时通过自动组装和反向组装流程维护对各个组件的库存跟踪。

#### 主要特征

- **组件结构**：具有多个子组件的父套件
- **自动处理**：系统管理的组装和反向组装
- **双重库存影响**：组件减少，组装的套件增加
- **制造集成**：生产和组装工作流程
- **质量控制**：组件验证和确认

#### 组件管理

**组装流程**：
1. 从库存中选择套件组件
2. 系统验证组件可用性
3. 组件库存水平减少
4. 组装的套件库存增加一个单位
5. 成本计算包括所有组件成本

**反向组装流程**：
1. 套件被拆解回组件
2. 套件库存水平减少
3. 各个组件库存水平增加
4. 成本核算反向恢复到组件价值

**套件组件**：
- **A1**：Intel i7 处理器（$300）
- **A2**：16GB DDR4 内存（$120）

- Kit-A 库存增加 1

### 3. 分组项目 (Grouped Item)

**定义**：分组项目管理具有尺寸或变体（尺寸、颜色、款式）的产品，同时保持单一产品标识，并为客户提供多种选择选项。

#### 主要特征

- **维度管理**：尺寸、颜色、款式和其他属性变体
- **客户选择**：交互式选择呈现
- **比例约束**：每笔交易仅选择一个基本项目
- **统一呈现**：具有多个选项的单一产品
- **库存跟踪**：对每个变体进行单独跟踪

#### 分组结构

**父组**：T 恤产品线
- **颜色变体**：黄色、蓝色、红色
- **尺寸变体**：小号 (3)、中号 (4)、大号 (5)
- **客户选择**：一种颜色 + 一种尺寸 = 一个基本 FI-item

#### 客户体验流程

1. **产品选择**：客户从目录中选择"T 恤"
2. **选项呈现**：系统显示颜色和尺寸选项
3. **客户选择**：选择"蓝色"颜色和"中号"尺寸
4. **项目解析**：系统识别特定的基本 FI-item
5. **交易处理**：为所选变体处理销售

### 4. 套餐项目 (Package Item)

**定义**：套餐项目建立一对多关系，其中单个 FI-item 映射到多个 INV items，实现具有灵活库存管理的复杂产品捆绑。

#### 主要特征

- **多重映射**：一个 FI-item 包含多个 INV items
- **延迟扣减**：仅在套餐销售时减少库存
- **灵活组成**：各种项目的不同数量
- **套餐定价**：整个套餐的单一价格
- **促销用途**：特别优惠和捆绑交易

#### 套餐行为

**销售前库存**：
- 组件项目保留在各自的库存池中
- 套餐创建期间没有库存移动
- 在套餐售出之前，组件可单独销售

**套餐销售时**：
- 所有组件 INV items 根据套餐规格减少
- 单笔交易处理多个库存移动
- 财务过账反映套餐价格分配

#### 套餐组成示例

**办公入门套餐**：
- 5x 钢笔 (INV-001)
- 3x 笔记本 (INV-002)
- 1x 订书机 (INV-003)
- 1x 胶带座 (INV-004)
- **套餐价格**：$25.00（相比单独总价 $32.00）

**软件捆绑套餐**：
- 1x 操作系统许可证 (INV-501)
- 1x 办公套件许可证 (INV-502)
- 1x 杀毒软件许可证 (INV-503)
- **套餐价格**：$199.00（相比单独总价 $300.00）

### 5. 捆绑项目 (Bundle Item)

**定义**：捆绑项目代表最复杂的项目类型，能够以分层结构包含基本项目、套件、分组项目或套餐的任意组合，最多支持两个层级的复杂性。

#### 主要特征

- **通用容器**：可以包含任何其他项目类型
- **多个 GenDocLines**：复杂的交易行生成
- **两级支持**：具有子捆绑能力的主捆绑
- **灵活定价**：单独或捆绑定价策略
- **高级场景**：企业级产品组合

#### 捆绑复杂性级别

**第 1 级捆绑**：各种项目类型的直接组合
- 基本项目 + 套件 + 分组项目 + 套餐
- 具有直接处理的单级结构
- 适用于大多数业务场景

**第 2 级捆绑**：嵌套捆绑结构
- 包含子捆绑的主捆绑
- 具有多个处理层的复杂层级
- 高级企业场景

#### 捆绑组成示例

**企业 IT 解决方案捆绑**：

**主捆绑**：完整办公设置
- **硬件套件**：计算机、显示器、键盘（套件项目）
- **软件套餐**：操作系统和应用程序（套餐项目）
- **基本项目**：电缆、适配器、电涌保护器（基本项目）
- **服务组**：安装选项（分组项目 - 现场/远程/标准/高级）

**嵌套子捆绑**：网络基础设施
- **网络套件**：路由器、交换机、电缆（套件项目）
- **安全套餐**：防火墙软件和许可证（套餐项目）
- **基本项目**：网络配件（基本项目）

#### 交易处理复杂性

**多个 GenDocLines 生成**：
- 每个捆绑组件生成单独的文档行
- 复杂的成本分配和定价计算
- 捆绑上下文中的单个组件跟踪
- 用于捆绑分析的详细报告

## 选择正确的项目类型

### 决策框架

使用此框架为您的业务场景确定适当的项目类型：

#### 从基本问题开始

1. **单个还是多个项目？**
   - 单个 → 考虑基本项目
   - 多个 → 评估其他类型

2. **库存关系？**
   - 一对一 → 基本项目
   - 一对多 → 套餐、套件或捆绑
   - 多对一 → 分组项目

3. **组件处理？**
   - 无需处理 → 套餐
   - 需要组装 → 套件
   - 复杂组合 → 捆绑

4. **客户体验？**
   - 简单选择 → 基本项目
   - 选项选择 → 分组项目
   - 捆绑选择 → 套餐或捆绑

## 与其他模块集成

### 财务会计集成

**成本核算**：
- 基本项目：直接成本分配
- 套件：组件成本累积
- 分组项目：变体特定成本核算
- 套餐：分配的成本分布
- 捆绑：复杂的成本层级管理

- 项目类型决定收入过账方法
- 捆绑组件可能需要单独的收入流
- 套餐销售使用合并收入方法

**库存跟踪**：

**采购集成**：

- 套件的组件可用性检查
- 套餐组件采购规划

### 销售和 CRM 集成
