## 💡 第 4 部分：核心概念

### 了解 BigLedger 架构

{{< callout type="info" >}}
**关键概念**：BigLedger 采用模块化架构，其中 **Applet** 是可跨多个**模块**（业务领域）使用的功能组件。
{{< /callout >}}

### 基本术语

| 术语 | 定义 | 示例 |
|------|------------|---------|
| **租户** | 您的公司实例 | demo-v1 |
| **模块** | 业务功能领域 | 销售、库存 |
| **Applet** | 可重用组件 | 税务配置 |
| **单据** | 业务交易 | 发票、采购订单 |
| **工作流** | 自动化流程 | 审批链 |
| **主数据** | 核心业务数据 | 客户、物料 |

## 🚀 第 5 部分：您的第一笔交易

### 创建示例销售订单

{{< tabs items="步骤 1：导航,步骤 2：创建,步骤 3：处理,步骤 4：验证" >}}
  {{< tab >}}
  **进入销售订单**

  1. 点击主菜单中的**销售**
  2. 选择**销售订单**
  3. 您将看到现有订单列表
  4. 点击**新建销售订单**按钮（右上角）
  {{< /tab >}}

  {{< tab >}}
  **输入订单详情**

  1. **客户部分**
     - 搜索：输入 "ACME"
     - 选择："ACME Corporation"
     - 注意地址自动填充

  2. **添加行项目**
     - 点击"添加物料"
     - 搜索："PROD-001"
     - 数量：10
     - 价格自动填充

  3. **订单信息**
     - 交货日期：明天
     - 付款条款：Net 30
     - 备注："演示订单"
  {{< /tab >}}

  {{< tab >}}
  **保存并处理**

  1. 点击**保存草稿**
     - 生成订单编号
     - 状态显示为"草稿"

  2. 点击**提交审批**
     - 状态变更为"待处理"
     - 发送通知

  3. 点击**批准**（作为管理员）
     - 状态："已批准"
     - 可供履行
  {{< /tab >}}

  {{< tab >}}
  **验证您的工作**

  1. **检查订单列表**
     - 您的订单出现在顶部
     - 状态显示正确

  2. **查看订单详情**
     - 点击订单编号
     - 查看所有信息

  3. **检查活动日志**
     - 查看创建时间戳
     - 查看审批历史
  {{< /tab >}}
{{< /tabs >}}

## 📊 第 6 部分：基本报表

### 生成您的第一份报表

1. **导航至报表**
   - 点击主菜单中的"报表"
   - 选择"销售报表"

2. **选择报表类型**
   - 选择"销售汇总"
   - 设置日期范围：本月

3. **配置选项**
   - 分组依据：客户
   - 包括：所有产品
   - 格式：PDF

4. **生成并导出**
   - 点击"生成报表"
   - 预览结果
   - 按需导出
