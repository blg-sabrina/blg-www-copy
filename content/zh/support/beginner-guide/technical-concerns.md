## 数据安全与隐私

**问：我的数据在云端安全吗？**

**答：** 这是最重要的问题之一！您的业务数据在 BigLedger 的云端实际上比在您办公室的电脑上更安全：

**为什么云存储更安全**：

**专业安全基础设施**：
- **专职安全团队**：全职专家保护您的数据
- **企业级防火墙**：军用级保护
- **多个数据中心**：您的数据存在于多个安全位置
- **全天候监控**：持续监视威胁

**自动备份保护**：
- **多个副本**：您的数据自动复制到多个位置
- **实时备份**：每次更改都会立即保存
- **地理分布**：副本存储在不同城市/国家
- **版本历史**：可以从任何时间点恢复数据

**高级加密**：
- **银行级加密**：与网上银行相同的安全性
- **加密传输**：数据在往返服务器时受到保护
- **加密存储**：即使有人访问了服务器，数据也是加密的
- **密钥管理**：只有授权人员才能解密您的数据

**与典型办公室设置的比较**：

| 您的办公室电脑 | BigLedger 云端 |
|---------------------|-----------------|
| 一台电脑 = 一个故障点 | 多个冗余系统 |
| 手动备份（常常被遗忘） | 自动持续备份 |
| 基本杀毒保护 | 企业级安全基础设施 |
| 物理盗窃风险 | 数据不在物理现场 |
| 火灾/水灾脆弱性 | 地理分布 |
| 想起来才更新 IT | 自动安全更新 |

**真实世界的安全效益**：

*场景 1：办公室火灾*
- **您的电脑**：数据完全丢失
- **BigLedger**：从任何电脑访问您的数据，无任何损失

*场景 2：笔记本电脑被盗*
- **本地文件**：小偷可以访问您的业务数据
- **BigLedger**：数据在云端安全，更改密码，继续工作

*场景 3：电脑病毒*
- **本地系统**：可能损坏或删除文件
- **BigLedger**：您的数据不受影响，从干净的电脑访问

**合规与认证**：
- **ISO 27001**：国际安全标准认证
- **SOC 2**：经审计的安全控制
- **符合 GDPR**：符合欧洲隐私法
- **符合 PDPA**：马来西亚个人数据保护

**您对数据安全的控制**：
- **强密码**：选择安全、唯一的密码
- **双重身份验证**：额外的登录安全层
- **用户权限**：控制谁可以访问什么
- **定期审查**：监控谁有权访问您的数据

**业务连续性规划**：

**为中断做准备**：
- **下载关键报告**：每周备份关键报告
- **移动应用设置**：确保团队已配置移动访问
- **备用地点**：确定附近有互联网的地方

**关键操作**：

- **多个提供商**：使用不同的提供商作为备用
- **云端优势**：只要有互联网，随处可访问数据
- **快速恢复**：大多数中断在数小时内解决

**真实世界影响评估**：

*典型小企业影响*：
- **短暂中断（1-2 小时）**：影响最小，使用移动数据或等待
- **中等中断（半天）**：使用移动数据，从其他地点工作
- **长时间中断（全天）**：影响显著但通过规划可管理

*您可以维持的关键功能*：
- **客户服务**：电话和面对面服务
- **销售**：手动收据，连接恢复后录入数据

**问：我如何与我的银行集成？**

**答：** 银行集成比您想象的更简单，BigLedger 支持多种连接方法：

**银行集成类型**：

**直接 API 连接**（首选方法）：
- **实时连接**：交易立即出现
- **自动导入**：无需手动工作
- **安全身份验证**：银行批准的连接方法
- **只读访问**：BigLedger 只能查看，不能转移资金

**文件导入方法**（通用备用）：
- **从银行下载**：从网上银行导出交易
- **上传到 BigLedger**：拖放文件导入
- **自动处理**：系统识别银行格式
- **手动分类**：审查并分类交易

**银行数据源服务**（折中方案）：
- **第三方服务**：Yodlee、Plaid 或类似服务
- **多银行支持**：通过一项服务连接多家银行
- **标准化格式**：无论哪家银行都提供一致的数据
- **可靠连接**：专业公司维护连接

**设置银行集成**：

**步骤 1：选择集成方法**
- 检查您的银行是否支持直接 API
- 评估文件导入作为备用选项
- 考虑为多个账户使用银行数据源服务

**步骤 2：安全授权**
- **API 方法**：通过银行网站授权 BigLedger
- **文件方法**：设置网上银行访问
- **数据源服务**：在数据源提供商处创建账户

**步骤 3：配置导入设置**
- **账户映射**：将银行账户与 BigLedger 账户匹配
- **交易规则**：设置自动分类
- **重复检测**：防止两次导入同一笔交易

**步骤 4：测试与验证**
- **导入测试期**：从近期交易开始
- **验证准确性**：检查金额和日期是否匹配
- **分类审查**：确保正确的交易分类

**支持的马来西亚银行**：
- **Maybank**：提供完整 API 集成
- **CIMB**：直接连接和文件导入
- **Public Bank**：文件导入和网上银行导出
- **Hong Leong**：多种连接方法
- **RHB**：支持标准文件格式
- **AmBank**：网上银行集成
- **Bank Islam**：符合伊斯兰教法的连接方法

**安全考量**：
- **只读访问**：集成无法转移或花费资金
- **银行授权**：您控制 BigLedger 可以访问什么
- **安全协议**：所有连接使用银行级加密
- **审计追踪**：所有数据访问的完整记录

**银行集成的好处**：
- **节省时间**：数据录入时间减少 80%
- **提高准确性**：消除手动录入错误
- **实时现金流**：始终了解您当前的现金状况
- **自动对账**：自动匹配交易
